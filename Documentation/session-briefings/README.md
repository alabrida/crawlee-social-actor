# Session Briefings

This directory contains one markdown file per completed session. Each briefing follows a standardized format so that a new agent session can quickly onboard.

## Naming Convention

`session-YYYY-MM-DD-NN.md` where `NN` is a sequence number if multiple sessions occur on the same day.

## Required Sections

Every briefing must include:

1. **Header** — Sprint number, platforms worked, session duration
2. **Accomplished** — Checklist of completed items
3. **In Progress** — Any partially completed work
4. **Next Session Should** — Numbered list of exact next steps
5. **Open Blockers** — Current blocker status
6. **Files Changed** — List of all files created or modified
